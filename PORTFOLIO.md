## Interview Talking Points

### Inbox Scout

- I built a Python automation tool that connects Gmail, Telegram, and AI classification into a guarded email cleanup workflow.
- The project emphasizes safety: no permanent delete, protected categories, audit logs, and reversible Gmail Trash actions.
- It shows I can work with APIs, OAuth, local/cloud AI routing, and real operational constraints.

### Remote Support Operations Lab

- I created realistic support tickets and knowledge base articles to show how I would communicate with users and document issues.
- The lab includes scenarios like VPN issues, Microsoft 365 access, phishing reports, slow computer performance, and Azure VM connectivity.
- It demonstrates that I understand support work is both technical troubleshooting and clear communication.

### Windows Command-Line Troubleshooting Lab

- I used real Windows commands to gather troubleshooting evidence and explain what each command proves.
- The project shows comfort with `ipconfig`, `ping`, `tracert`, `nslookup`, `systeminfo`, `tasklist`, and `netstat`.
- I also redacted sensitive output, showing awareness of privacy and documentation safety.

### Active Directory on Azure Lab

- I built a basic domain lab using Azure VMs, Windows Server, Active Directory, DNS, and a Windows client.
- I practiced common support tasks such as user creation, account unlocks, password resets, disabling/enabling users, and domain join concepts.
- It demonstrates the foundation needed for help desk and junior systems roles.

### Azure Network Protocols and NSG Lab

- I used Wireshark and command-line tools to observe network protocols and test Azure NSG rules.
- The lab shows how I troubleshoot connectivity by checking protocol behavior, firewall rules, and expected vs actual results.
- It connects cloud fundamentals with practical support troubleshooting.

### osTicket Help Desk Series

- I deployed osTicket, configured support operations, and documented ticket lifecycle workflows.
- The series shows I understand how help desks organize agents, departments, help topics, assignments, and ticket resolution.
- It demonstrates both platform setup and day-to-day support process awareness.
